# 随机选择决策工具 — 设计文档

## 1. 定位

嵌入当前项目「效率工具」分类，复用 `pages/wheel/wheel` 页面，单页面承载三种随机决策模式：**骰子**、**转盘**、**抽签**。

三种模式共用同一套选项数据，切 Tab 不丢数据。

## 4. 三种模式设计

### 4.1 转盘模式 (Canvas 2D)

**渲染**：
- 所有选项均分 360° 扇形区域
- 自动配色：预定义 8 色调色板，循环使用，相邻不同色
- 文字沿扇形中线径向排列，字号自适应扇区大小
- 顶部固定指针（三角形指示器）

**交互**：
- 点击「开始」→ 高速旋转 + 缓停（easing: cubic-bezier）
- 旋转过程约 2-3 秒，先快后慢自然停止
- 指针所指扇区 = 结果
- 期间按钮禁用，动画结束后弹结果 + 短震动

**容错**：选项 ≤1 个时，不旋转直接弹结果

**Canvas 方案**：
- `requestAnimationFrame` 驱动旋转动画
- 每帧绘制：扇区底色 → 扇区文字 → 中心圆 → 指针（固定不动）
- 旋转角度增量递减模拟缓停

### 4.2 骰子模式 (CSS 3D / Canvas)

**逻辑**：
- ≤6 个选项：显示传统 6 面骰子，每面绘制对应选项文字，不足 6 面留白
- >6 个选项：骰子显示数字 → 映射到选项列表下标

**交互**：
- 点击骰子 → 3D 旋转抖动动画 → 随机停在一面 → 弹结果 + 震动

**实现**：
- Canvas 绘制 2D 骰子正面（简化方案，不做真 3D）
- 快速轮播几个面 → 最终定格
- 或使用 CSS 3D transform 做真立方体旋转

### 4.3 抽签模式 (Canvas)

**视觉**：
- 垂直排列签条（竹简风格），每签显示选项文字
- 签条轻微随机摇摆/浮动

**交互**：
- 手机模拟摇晃（监听加速度）或点击「摇签」按钮
- 签条乱序晃动 → 一支签弹出置顶 → 展示结果

**实现**：
- Canvas 绘制签条列表
- 动画：随机 Y 偏移抖动 → 选中签高亮上移 → 结果文字放大

## 5. 交互流程

```
进入页面
  │
  ├─ 首次 → 加载默认「吃什么」选项
  ├─ 非首次 → 加载本地缓存选项
  │
  ├─ [快捷场景] → ActionSheet → 选场景 → 替换选项 → toast 提示
  ├─ [选项管理] → 面板展开 → 增删改选项 → 实时生效
  ├─ [+ 添加] → 新增空行 → 输入文字
  │
  └─ 选模式 Tab → 点击随机区域 / 「开始」按钮
        │
        ├─ 过滤空选项
        ├─ <2 个？ → 弹提示「至少两个选项」
        └─ ≥2 个？ → 动画 + 震动 → 弹结果
```

## 6. 技术要点

### Canvas 转盘渲染

```
扇形角度 = 360° / 选项数
for each 选项 i:
  起始角 = i * 扇形角度 - 90°（从顶部开始）
  填充色 = PALETTE[i % 8]
  ctx.arc + ctx.fill()
  ctx.save → rotate(中点角) → fillText(文字)
文字沿径向排列，字号 = min(扇形弧长/文字宽度, 最大字号)
```

### 缓停动画

```
旋转总时长: 2.5s
初始速度: 720°/s
缓动函数: cubic-bezier(0.17, 0.67, 0.12, 0.99)
最终随机角度: Math.random() * 360
总旋转角度: 5*360 + 最终随机角度（至少转5圈）
```

### 数据持久化

```
storage key: 'random_picker_options'
存储：选项数组
读取：onLoad → 有缓存用缓存，无缓存用默认
写入：每次选项变更时 setStorageSync
```
